机器学习策略（上）

1. Why ML Strategy

当我们最初得到一个深度神经网络模型时，我们可能希望从很多方面来对它进行优化，例如：

  • Collect more data

  • Collect more diverse training set

  • Train algorithm longer with gradient descent

  • Try Adam instead of gradient descent

  • Try bigger network

  • Try smaller network

  • Try dropout

  • Add L2 regularization

  • Network architecture: Activation functions, #hidden units…

可选择的方法很多，也很复杂、繁琐。盲目选择、尝试不仅耗费时间而且可能收效甚微。因此，使用快速、有效的策略来优化机器学习模型是非常必要的。

2. Orthogonalization

正交化

如果想要调整某个参数来优化这个模型

控制变量才是最好最容易控制的

改变某个参数能够只是改变这个模型的特定的功能能力才是最好的

也就是一种参数对应一种能力

而不是一个参数多个能力发生改变

  • Fit training set well on cost function
  • Fit dev set well on cost function
  • Fit test set well on cost function
  • Performs well in real world

这些是常见的需要调参的地方

3. Single number evaluation metric

同一个功能的若干个不同的模型，可能有多个方面的优劣

想要选一个好的，就需要一个评价指标来对比

$F1$的出现就可以方便评估模型的优劣

这只是一种算法，算法是可以变得，所以不用管它怎么来的

4. Satisficing and Optimizing metic

有时把所有的指标全部用一个算法满足是很难的

方便起见就有了满意指标和优化指标

  • 满意指标：只要达到了阈值就行
  • 优化指标：要尽可能的越优秀越好

假设100ms以内用户不会在意这个分类器的快慢了

所以阈值达到就好了

accuracy越优秀越好，

所以选择B

5. Train/dev/test distributions

验证集和测试集应该在同一分布才能有更好的效果

6. Size of the dev and test sets

在之前的课程中我们已经介绍过，当样本数量不多（小于一万）的时候，通常将Train/dev/test sets的比例设为60%/20%/20%，在没有dev sets的情况下，Train/test sets的比例设为70%/30%。当样本数量很大（百万级别）的时候，通常将相应的比例设为98%/1%/1%或者99%/1%。

  • 对于dev sets数量的设置，应该遵循的准则是通过dev sets能够检测不同算法或模型的区别，以便选择出更好的模型。

  • 对于test sets数量的设置，应该遵循的准则是通过test sets能够反映出模型在实际中的表现。

实际应用中，可能只有train/dev sets，而没有test sets。这种情况也是允许的，只要算法模型没有对dev sets过拟合。但是，条件允许的话，最好是有test sets，实现无偏估计

9. Avoidable bias

过拟合和欠拟合的评价指标

  • bias:人的错误率和test的错误率的差值
  • variance:train的错误率和test的错误率的差值

比较这两个指标可以判断是过拟合还是欠拟合

1. Carrying out error analysis

我们要对提高正确率的因素进行分析

怎么样能提高正确率，减小错误率才是最重要的

少做无用功

  • Fix pictures of dogs being recognized as cats
  • Fix great cats(lions, panthers, etc…) being misrecognized
  • Improve performance on blurry images

2. Cleaning up incorrectly labeled data

要不要进行这个操作和上面一样，取决于这个因素到底是不是主要因素

如果不是主要因素，就让他

如果是主要因素，就改

3. Build your first system quickly then

  • Set up dev/test set and metric
  • Build initial system quickly
  • Use Bias/Variance analysis & Error analysis to prioritize next steps

4. Training and testing on different distribution

如果训练集和测试集的区别很大（也就是不同的分布）怎么办

有两种方法

  • 将两个分布的的数据集完全混合，然后随机抽取一部分数据集当作训练集，一部分测试集（不是很好）

  • 将一部分测试集的数据加入到训练集之中，然后剩下的那一部分用来测试（挺好的）

5. Bias and Variance with mismatched data distributions

另一种数据集train-dev，是和train同一个部的数据集，只不过train-dev部分用来进行测试而已

这个数据集所体现的是和train的比较，如果差距很大那就欠拟合

6. Addressing data mismatch

有两个建议

  • 手动来解决训练集和测试集的样本的不同
  • 将测试集分布的样本放到训练集里面进行训练

对于第二个点不能放太多，因为放太多了会影响训练的效果，增加训练的难度，导致噪声的过拟合从而没有噪声的反而识别不出来了。

7. Transfer learning

迁移学习本质就是将原来的神经网络拆分成不同的功能部分，比如卷积神经网络，前面是提取特征，后面才是分类，所以可以迁移前面的与训练模型，然后用于不同的分类任务。

迁移学习的三种常见应用场合：

  1. 任务A和B有相同的输入x：指的是任务A和B都是针对相同类型的输入数据进行处理和学习的，例如两个计算机视觉任务都是基于图像输入数据的处理。在这种情况下，可以将在任务A中学习到的知识和模型参数迁移到任务B中，以加速任务B的学习和提高其性能表现。
  2. 你拥有大量任务A的数据，但是任务B的数据相对较少：这种情况下，由于缺乏足够的任务B数据，可能很难直接训练一个有效的模型。此时，可以通过迁移学习，将在任务A中学到的模型参数和知识迁移到任务B中，以充分利用已有的数据资源，提高模型性能。
  3. 任务A中的低层特征可能有助于学习任务B：在一些任务中，输入数据包含许多共享的低层特征，例如图像中的边缘、纹理等，这些特征可以被任务A学习到并被迁移到任务B中，从而帮助模型更好地理解和处理任务B。

9. What is end-to-end deep learning

端到端的学习就是将之前每一步的功能都分的很细的神经网络用一个单一的神经网络给直接整合，也就是输入端到输出端中间有个神经网络，下面是语音识别中传统算法流程和端到端算法流程的区别

端到端的算法流程的好处

  • 如果神经网络足够大，效果会比一般的算法要好
  • 并且作为一个整体，各个权重之间耦合度非常高

10. Whether to use end-to-end deep learning

我们主要是根据优点和坏处来进行权衡

优点:

  • 不需要进行组件的设计
  • 直接让模型自己来提取特征

缺点：

  • 要大量的数据堆叠才能效果好
  • 可能用了组件效果会更好，然而你没用，那就不行咯
